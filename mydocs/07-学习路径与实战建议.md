# 07 学习路径与实战建议

## 第一阶段：先把项目跑起来（1-2 天）
1. 安装 Python 3.11 与 `uv`。
2. 执行 `uv sync --group dev` 安装依赖。
3. 了解两个主要入口：
   - GUI 入口：`src/zzz_od/gui/app.py`
   - 应用入口：`src/zzz_od/application/zzz_application_launcher.py`
4. 先只改配置文件，不改核心代码。

## 第二阶段：看懂主链路（3-5 天）
1. 控制链：`pc_controller_base.py` -> `zzz_pc_controller.py`
2. 识别链：`onnx_ocr_matcher.py` + `src/onnxocr/*` + `src/one_dragon/yolo/*`
3. 编排链：`Operation` / `Application` / `ApplicationRunContext`
4. 业务链：`src/zzz_od/application/*` + `src/zzz_od/operation/*`

## 第三阶段：做一次小改动（约 1 周）
1. 选一个具体功能（如某个日常任务）。
2. 建议改动顺序：
   - 先改 `config/*.sample.yml`
   - 再改对应 `*_app.py` / `operation/*.py`
   - 最后改 GUI 显示层
3. 每次改一个点，改完立即验证。

## 第四阶段：补工程化能力（持续）
1. 学会读 `test-check.yml`（测试门禁）。
2. 学会读 `build-release.yml`（构建/签名/发布）。
3. 理解 `pyproject.toml` 中依赖和 Ruff 规则。

## 安全调试建议
1. 优先使用测试账号和低风险场景验证。
2. 先看日志，再看截图，再看阈值与输入尺寸。
3. 遇到输入异常先检查：窗口焦点、管理员权限、分辨率缩放。
4. 不要一开始修改底层控制器，先从配置和上层流程入手。

## 推荐最短阅读路线
1. `03` 中 `one_dragon/base/operation/*`
2. `03` 中 `one_dragon/base/controller/*`
3. `04` 中 `onnxocr/*`
4. `05` 中 `zzz_od/context/zzz_context.py` 与 `zzz_od/application/*`
